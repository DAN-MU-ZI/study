# 장애 시나리오

주요 장애 유형
1. 같은 주문에 대해 중복 승인 요청이 들어온다.
2. 대기 중인 요청이 선행 요청의 실패를 함께 받지 못하고 timeout으로 끝난다.
3. 같은 멱등성 키가 다른 payload와 함께 재사용된다.
4. 사용자가 다른 주문을 시작했는데도 프런트가 이전 키를 계속 사용한다.

advanced 실습이 막아야 하는 것
- Redis 락이 같은 키에서 여러 선행 요청이 생기지 않도록 막는다.
- Redis 결과 저장이 성공뿐 아니라 실패도 다시 재생한다.
- payload 일관성 검사가 같은 키에 다른 요청 본문이 섞이는 것을 막는다.
- 프런트는 새 주문이 시작될 때만 키를 교체한다.
