# 5장: 개선 구조 요구사항 정리

## 핵심 질문

레거시 실패 시나리오를 줄이려면 어떤 조건을 만족해야 하는지 정리한다.

## 요구사항 1: 예약 판단과 상태 변경의 트랜잭션 통합

예약 가능한 재고를 확인하는 작업과 예약 상태를 변경하는 작업이 같은 트랜잭션 안에서 처리되어야 한다. 그래야 중간 실패가 발생했을 때 롤백 기준이 명확해진다.

## 요구사항 2: 같은 재고 단위의 중복 예약 방지

동시에 여러 요청이 들어와도 같은 재고 1개를 두 요청이 함께 잡으면 안 된다. 재고 단위가 명확하게 식별되어야 한다.

## 요구사항 3: 락 대기 완화

하나의 요청이 특정 row를 잠갔다고 해서 다른 요청이 전체 예약 처리를 멈출 필요는 없다. 이미 잠긴 row는 건너뛰고 다른 예약 가능한 row를 가져올 수 있어야 한다.

## 요구사항 4: 예약 가능한 재고 pool 유지

모든 재고를 row로 만들면 데이터가 과도하게 커질 수 있다. 필요한 크기의 예약 가능한 row pool을 유지하고, 부족해지면 보충해야 한다.

## 요구사항 5: 만료와 확정 처리의 상태 수렴

예약 만료와 결제 성공이 경합해도 최종 상태가 하나로 수렴해야 한다. 같은 예약이 중복 차감되거나 중복 복구되면 안 된다.
