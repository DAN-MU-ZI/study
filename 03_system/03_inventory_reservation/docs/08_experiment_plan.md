# 8장: 실험 계획 수립

## 핵심 질문

문서로 정리한 구조가 실제 동시성 상황에서 어떻게 동작하는지 확인한다.

## 실험 1: Redis 레거시 예약

목표:

- Redis 기반 가용 수량 차감이 정상적으로 동작하는지 확인한다.
- 예약 TTL과 해제 흐름을 확인한다.
- Redis와 MySQL 원장의 트랜잭션 경계가 분리되는 지점을 확인한다.

기록 항목:

- Redis 가용 수량
- Redis 예약 상태
- MySQL 원장 수량
- 예약 로그 상태

## 실험 2: 레거시 실패 시나리오

목표:

- 중간 단계 실패를 주입한다.
- Redis와 MySQL의 최종 상태가 어떻게 어긋나는지 기록한다.

기록 항목:

- 실패 주입 위치
- Redis 최종 상태
- MySQL 최종 상태
- 오버셀 또는 언더셀 발생 여부
- 수동 복구 방법

## 실험 3: MySQL `SKIP LOCKED`

목표:

- 같은 pool을 대상으로 여러 트랜잭션이 동시에 예약을 시도한다.
- 잠긴 row를 대기하는 방식과 건너뛰는 방식을 비교한다.

기록 항목:

- 세션별 확보한 unit id
- 락 대기 여부
- 실패한 예약 수
- 최종 `reservation_units` 수
- 최종 `reserved_quantities` 수

## 실험 4: 보충 시스템

목표:

- 예약 가능한 row pool이 줄어든 상태를 만든다.
- pool 보충이 원장 수량을 초과하지 않는지 확인한다.
- 중복 보충 시도가 제어되는지 확인한다.

기록 항목:

- 보충 전 pool 크기
- 보충 후 pool 크기
- `next_unit_id` 변화
- 중복 unit id 발생 여부
- 원장 기준 가용 수량 초과 여부

## 실험 5: 구조 비교

목표:

- 같은 초기 조건에서 레거시 구조와 개선 구조를 비교한다.

기록 항목:

- 성공 예약 수
- 실패 예약 수
- 상태 불일치 수
- 락 대기 또는 대기 시간
- 복구 난이도
